- [ ] **Step 1.2.1: Create frontmatter stripping script**

```bash
cat > /tmp/strip-frontmatter.sh << 'SCRIPT'
#!/usr/bin/env bash
# Strip YAML frontmatter from a markdown file, output pure body.
# Usage: strip-frontmatter.sh <input> <body-output> [<frontmatter-output>]
# Handles leading blank line after closing --- (standard markdown convention).
set -euo pipefail
input="$1"; body_out="$2"; fm_out="${3:-/dev/null}"
fm_done=0; fm_count=0; skip_blank=0
> "$body_out"  # truncate output
while IFS= read -r line; do
    if [[ "$fm_done" -eq 0 ]] && [[ "$line" == "---" ]]; then
        fm_count=$((fm_count + 1))
        echo "$line" >> "$fm_out"
        if [[ "$fm_count" -ge 2 ]]; then
            fm_done=1
            skip_blank=1  # skip one blank line after closing ---
        fi
    elif [[ "$fm_done" -eq 0 ]] && [[ "$fm_count" -ge 1 ]]; then
        echo "$line" >> "$fm_out"
    elif [[ "$skip_blank" -eq 1 ]] && [[ -z "$line" ]]; then
        skip_blank=0  # consumed the blank line, don't output it
    else
        skip_blank=0
        echo "$line" >> "$body_out"
    fi
done < "$input"
SCRIPT
chmod +x /tmp/strip-frontmatter.sh
```

```markdown
## Self-Verification Gate (MANDATORY)

Before presenting ANY finding to the user or writing it to a verdict file:

1. For each finding that cites a file path and line number:
   - Read the cited file at the cited line
   - Verify the described issue actually exists at that location
   - If the file or line does not contain the described issue, DISCARD the finding

   - Grep for the function definition
   - Verify the function exists and the described behavior is accurate
3. Log: "Verified N/M findings (discarded M-N false positives)"

DO NOT present unverified findings. A smaller, verified list is always
better than a larger list containing false positives.
```

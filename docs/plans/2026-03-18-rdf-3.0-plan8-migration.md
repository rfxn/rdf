# RDF 3.0 Plan 8: Migration from RDF 2.x to 3.0

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the cutover from RDF 2.x to 3.0 by promoting v3 agents and commands to their canonical locations, updating the adapter system and CLI, refreshing documentation, and removing all obsolete v2 artifacts.

**Architecture:** Plans 1-7 developed v3 agents, commands, and infrastructure in parallel directories (`canonical/agents-v3/`, `canonical/commands-v3/`, `adapters/claude-code/agent-meta-v3.json`). This plan promotes those artifacts to production locations, archives the v2 originals, updates all tooling to reference the new structure, and bumps VERSION to 3.0.0. This is the riskiest plan in the series — every step must be verified before proceeding to the next, and archives must be confirmed before any deletions.

**Tech Stack:** Bash (CLI, adapter), JSON (metadata), Markdown (agents, commands, docs)

**Spec:** `docs/specs/2026-03-18-rdf-3.0-architecture-design.md` (Migration from RDF 2.x section)

**Risk:** HIGH — This plan replaces all production agents, commands, adapter metadata, and CLI logic in a single coordinated cutover. Incorrect ordering will break `rdf generate`.

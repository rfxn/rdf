# Governance Index Schema

## Purpose

The governance index is generated by `/r-init` and lives at
`.rdf/governance/index.md` in the target project. It is the ONLY
governance artifact that is always loaded into agent context.

## Budget

- Maximum 50 lines
- Target ~100-150 tokens
- Must survive context compaction

## Format

The index uses a fixed structure with one-line descriptions:

    # Governance Index

    ## Project
    - Name: {project-name}
    - Branch: {current-branch}
    - Mode: {operational-mode or "development"}
    - Plan: {phase-status or "none"}

    ## Authoritative Files
    - {filename} — {one-line description of what it covers}
    (one line per existing .md file ingested during init)

    ## Governance Files
    - architecture.md — {one-line summary}
    - conventions.md — {one-line summary}
    - verification.md — {one-line summary}
    - constraints.md — {one-line summary}
    - anti-patterns.md — {one-line summary}
    (only files that were generated; omit empty/unused ones)

    ## Reference
    - {filename} — {one-line description}
    (one line per reference doc copied from profile)

## Rules

1. One line per file, no multi-line descriptions
2. Authoritative Files section lists existing project .md files
   (CLAUDE.md, AGENTS.md, etc.) — these are NOT in .rdf/governance/
3. Governance Files section lists generated supplements in
   .rdf/governance/
4. Reference section lists profile reference docs in
   .rdf/governance/reference/
5. Agents read this index to decide which files to load just-in-time
